# 作业帮题单 PDF 题目裁剪器

将作业帮错题本 PDF 中的每道题目自动识别并裁剪为独立的图片文件，支持跨页题目自动合并。

## 适用场景

- 将作业帮错题本 PDF 按题目拆分为独立图片
- 从作业帮题单中提取单道题目用于制作课件或题库
- 用户需要将作业帮 PDF 试卷按题号拆分为单独的图片

## 触发条件

- 用户要求"裁剪PDF中的题目"、"拆分试卷"、"按题目分割PDF"
- 用户提到"把PDF每道题切出来"、"提取PDF中的每道题"
- 用户需要将作业帮题单按题号拆分为单独的图片

## 硬编码配置（作业帮专用）

本 skill 针对作业帮错题本 PDF 做了以下硬编码，无需每次重新分析：

| 配置项 | 值 | 说明 |
|--------|-----|------|
| 页面尺寸 | A4 (595×842) | 作业帮标准页面 |
| 顶部品牌区 | y=0~68 | logo 在 y≈52，第一题在 y≈80 |
| 底部页码区 | y=795~842 | 页码在 y≈802，图片底边在 y≈789 |
| 讲义页眉区 | y=0~68 | 日期\|知识点\|科目 + 提示 |
| 讲义页脚区 | y=809~842 | edsionc.top \| 邮箱 |
| 题目正则 | `第\d+题\|相似题\d+` | 作业帮主格式 |
| 封面特征 | `错题整理笔记\|掌握情况` | 自动跳过封面页 |

## 工作流程

### 第一步：环境准备

```bash
pip install pymupdf Pillow
```

### 第二步：运行裁剪脚本

```bash
python3 crop_questions.py <输入PDF> [输出目录]
```

如果不指定输出目录，默认在输入 PDF 同目录下创建 `<PDF名>_题目图片/` 文件夹。

脚本会自动完成以下工作：
1. **检测 PDF 类型**：自动识别是原始作业帮 PDF、已处理讲义、还是通用 PDF
2. **跳过封面页**：含"错题整理笔记"/"掌握情况"的页面自动跳过
3. **去除页眉页脚/品牌**：根据 PDF 类型应用对应的硬编码裁剪区域
4. **识别题目序号**：使用 `get_text("dict")` 的 line/span 级别提取，合并同行 span 后匹配
5. **全局标记排序**：把所有非封面页的题目序号按 `(page_index, y)` 全局排序
6. **智能分割与跨页合并**：对每个题号，跨页找下一个题号，通过"下一页首个题号离页眉距离"判断是否跨页：
   - gap < 80px：下一页开头就是新题，当前题不跨页
   - gap ≥ 80px：当前题内容延续到下一页，执行部分页拼接（当前页底部 + 中间整页 + 下一页顶部到下一题号）
7. **内容边界保护**：对不跨页且同页无下一题的题，扫描文本找实际内容底部；若内容底部离页底 <120px，保守截到页底，避免底部图片/绘图被切。
8. **识别知识点**：从全文匹配学科知识点字典，写入报告
9. **生成报告**：输出 `裁剪报告.txt`，含题目数量、来源页码、跨页合并信息、知识点

### 第三步：检查结果

裁剪完成后检查 `裁剪报告.txt`：
- 题目总数是否符合预期
- 是否有跨页合并的题目
- 是否有未识别到序号的题目
- 知识点识别是否准确

## 自动 PDF 类型检测

脚本会自动检测 PDF 类型并应用对应的裁剪策略：

| 类型 | 检测特征 | 顶部裁剪 | 底部裁剪 |
|------|---------|---------|---------|
| 已处理讲义 | 含"错题整理笔记"/"掌握情况" | 68px (页眉) | 33px (页脚) |
| 原始作业帮 | 含"作业帮"文字 | 68px (logo) | 47px (页码) |
| 通用 | 以上都不匹配 | 0px | 自动检测页脚 |

## 题目序号识别

### 主格式（作业帮专用）

```python
QUESTION_PATTERN = re.compile(
    r'^(?:'
    r'第\d+题'                              # 作业帮主格式: 第1题
    r'|相似题\d+'                           # 作业帮相似题: 相似题1
    r'|\d+[\.、．]'                         # 通用: 1. 1、 1．
    r'|[一二三四五六七八九十百]+[、．\.]'    # 通用: 一、二、
    r')'
)
```

### 小问序号（排除）

以下格式是题目内的小问序号，**不作为独立题目分割点**：

- `(1)`、`(2)`、`（1）`、`（2）` — 括号数字
- `①`、`②`、`③` — 带圈数字

### 关键：使用 dict 模式提取

**必须使用 `get_text("dict")` 的 line/span 级别提取**，不要使用 `get_text("blocks")`。

原因：`get_text("blocks")` 会把"第X题"标题和紧随其后的正文拆成独立的 block，导致每道题被错误拆成标题+正文两块。使用 dict 模式可以将同一行的多个 span 合并后再匹配。

## 已知问题与踩坑记录

### 问题1：文本块粒度导致过度拆分

**现象**：每道题被拆成"标题"和"正文"两块，导致题目数量翻倍。
**原因**：`get_text("blocks")` 把"第X题"（独立文本行）和紧随其后的正文拆成不同 block。
**修复**：改用 `get_text("dict")` 的 line/span 级别提取，将同一行的多个 span 合并后再匹配序号。

### 问题2：小问序号被误识别为题目序号

**现象**：题目内的小问 `(1)求证...`、`①聚乙烯可制成...` 被当作新题目的序号，导致题目被错误拆分。
**原因**：正则中包含了 `\(\d+\)`、`[①②③]` 等模式。
**修复**：从主正则中移除这些模式，并增加 `SUBQUESTION_PATTERN` 双重保险排除。

### 问题3：封面页"第X题"被误识别

**现象**：已处理讲义的封面页含"第1题~第15题"的掌握情况打勾表，被当作题目分割。
**原因**：封面页的打勾表中有"第X题"文字。
**修复**：增加 `COVER_PATTERN` 检测封面页特征（"错题整理笔记"/"掌握情况"），自动跳过。

### 问题4：页面开头空白块

**现象**：页面顶部到第一个题目序号之间产生一个空白或极小的裁剪块。
**修复**：对第一块（i==0）进行文本检测，如果没有文本内容则跳过。

### 问题5：分割线策略过于复杂

**现象**：原设计使用"相邻序号 Y 坐标中点"加"序号上方留余量"的分割策略，计算复杂且容易产生偏差。
**修复**：直接用序号的 Y 坐标作为分割点，上一题到此结束、下一题从此开始，简洁准确。

### 问题6：跨页题目判断错误

**现象**：第7题、第14题等靠近页底的题目，被错误地拼接了下一页的新题内容；而第4题、第11题等真正跨页的题目，有时续页丢失。
**原因**：旧逻辑只看"下一页首块是否有题号"就合并，无法区分"新题在页顶"和"当前题续页在页顶"。
**修复**：采用全局标记序列策略，通过"下一页首个题号离页眉的距离"判断（阈值 80px）。距离小 → 新题；距离大 → 当前题续页。

### 问题7：底部题目图片被切

**现象**：第7题、第14题等位于页面底部且带图的题目，图片被截掉一半。
**原因**：按文本内容底部裁剪时，`pymupdf` 文本提取拿不到图片/绘图位置，导致 bottom 停在文本末尾，下方图片丢失。
**修复**：当检测到的文本内容底部离页底 <120px 时，保守截到内容区底部，把图片也包含进来。

### 问题8：最后一题截取整页空白

**现象**：最后一题下方没有题号，旧逻辑直接截到页面底部，包含大量空白。
**原因**：没有检测实际内容边界。
**修复**：用 `find_content_bottom()` 扫描文本 spans 找实际内容底部，只在内容下方留 30px 空白。

## 注意事项

- 对于扫描版 PDF（纯图片无文字层），需要额外安装 OCR 工具（如 `pytesseract` + Tesseract-OCR），并在文本提取环节使用 OCR 代替 `pymupdf` 的文本提取。
- 题目序号正则默认排除了小问序号格式。如果目标 PDF 确实使用 `(1)` 或 `①` 作为独立题目编号，需要按需调整 `QUESTION_PATTERN`。
- 跨页判断阈值（80px）和内容边界阈值（120px）基于 A4 作业帮题单调试，若页面尺寸或排版差异较大，可适当调整。
- 输出图片为 PNG 格式（300 DPI），保证清晰度。如需 PDF 格式输出，可修改保存逻辑。
- 裁剪完成后务必检查报告中的题目总数是否符合预期，如有偏差需回到分析步骤排查。
